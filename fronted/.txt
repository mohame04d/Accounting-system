1️⃣ الشاشة الرئيسية (Dashboard)

المستخدم أول ما يفتح التطبيق لازم يشوف لوحة تحكم فيها أهم الأقسام:

أزرار الوصول السريع لكل وظيفة:

الوارد (Purchases)

الصادر / المبيعات (Sales)

الأصناف / المخزون (Items / Inventory)

الموردين (Suppliers)

تقارير / جرد

ملخص سريع:

آخر وارد تم إضافته

آخر مبيعات

عدد الأصناف

المخزون الحالي

ملفات Frontend المقترحة:

frontend/
├─ index.html        ← شاشة Dashboard الرئيسية
├─ index.js          ← منطق Dashboard
├─ style.css         ← تصميم عام لجميع الصفحات

2️⃣ شاشة الوارد (Purchases)

هنا المستخدم يقدر:

إضافة وارد جديد

رؤية جميع الواردات السابقة

تعديل أو حذف وارد (اختياري)

ملفات Frontend المقترحة:

frontend/
├─ purchase.html     ← فورم إضافة الوارد
├─ purchase.js       ← منطق التعامل مع الفورم و SQLite

3️⃣ شاشة الصادر / المبيعات (Sales)

مماثلة للوارد، فيها:

إضافة مبيعات جديدة

رؤية كل المبيعات

حساب إجمالي المبيعات، الأرباح، الخصومات

ملفات Frontend المقترحة:

frontend/
├─ sales.html
├─ sales.js

4️⃣ شاشة الأصناف / المخزون (Items / Inventory)

هنا المستخدم يقدر:

إضافة أصناف جديدة

تحديث الكميات والأسعار

عرض كل الأصناف مع الكمية الحالية

البحث عن صنف معين

ملفات Frontend المقترحة:

frontend/
├─ items.html
├─ items.js

5️⃣ شاشة الموردين (Suppliers)

هنا المستخدم يقدر:

إضافة مورد جديد

تعديل بيانات المورد

حذف مورد

ملفات Frontend المقترحة:

frontend/
├─ suppliers.html
├─ suppliers.js

6️⃣ شاشة التقارير / الجرد

عرض الجرد الشهري

تقارير الواردات والمبيعات

أرصدة المخزون

ملفات Frontend المقترحة:

frontend/
├─ reports.html
├─ reports.js

7️⃣ هيكل مجلدات Frontend كامل
frontend/
├─ index.html
├─ index.js
├─ style.css
├─ purchase.html
├─ purchase.js
├─ sales.html
├─ sales.js
├─ items.html
├─ items.js
├─ suppliers.html
├─ suppliers.js
├─ reports.html
├─ reports.js


ملاحظة: كل صفحة HTML تشترك في style.css عشان يبقى التصميم موحد.
والـ index.html (Dashboard) هي الصفحة اللي أول ما يفتح التطبيق تظهر للمستخدم.
